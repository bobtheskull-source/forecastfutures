API host vs GitHub Pages

The Pages client only renders market data and trade links. Credentials stay on the API host or local server, never in the browser.

  • Environment: KALSHI_BASE_URL
  • Auth secret: KALSHI_API_KEY
  • Private key path: KALSHI_PRIVATE_KEY_PATH
  • Keep Kalshi auth on the server side only.
  • Rotate any exposed key before production use.
  • Expose read-only market data to the mobile client.
  • Use simple retry handling and rate limits on the server edge.
  • Document the mobile/frontend Pages target separately from the API host.

Move rules
  • breakout candidate: move >= 12, volume >= 400, score floor 22
  • liquid mover: move >= 8, volume >= 700, score floor 18
  • watchlist drift: move >= 5, volume >= 1000, score floor 15
Edge cases
  • Tiny move plus huge volume can still be filtered out if the score stays below the floor.
  • Big move on thin volume is flagged, but the label should note lower confidence.
  • Zero or missing volume should never crash the detector, it should just score conservatively.
